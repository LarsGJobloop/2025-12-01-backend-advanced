# Technology References

Links to the various technologies we will use, grouped by when they are introduced:

## Week 1 - TDD

- [.NET 10.0](https://dotnet.microsoft.com/download/dotnet/10.0): Target framework for all projects
- [xUnit Testing Framework](https://xunit.net/?tabs=cs): Framework for writing tests in C# and .NET
- [ASP.NET Core Minimal APIs](https://learn.microsoft.com/en-us/aspnet/core/fundamentals/minimal-apis): Lightweight approach to building HTTP APIs using `app.MapGet`, `app.MapPost`, and `Results` types
- [Microsoft.AspNetCore.Mvc.Testing](https://learn.microsoft.com/en-us/aspnet/core/test/integration-tests): Package for integration testing with `WebApplicationFactory`
- [HttpClient](https://learn.microsoft.com/en-us/dotnet/api/system.net.http.httpclient): For making HTTP requests between services
- [IHttpClientFactory](https://learn.microsoft.com/en-us/dotnet/core/extensions/httpclient-factory): For managing HttpClient instances via dependency injection
- [Dependency Injection in ASP.NET Core](https://learn.microsoft.com/en-us/aspnet/core/fundamentals/dependency-injection): Built-in DI container for managing service dependencies

## Week 2 - Service-Oriented Architecture (SOA)

- [Docker Compose](https://docs.docker.com/compose/): Tool for defining and running multi-container Docker applications using a YAML file
- [Docker Compose File Reference](https://docs.docker.com/compose/compose-file/): Complete reference for compose.yaml syntax and options
- [PostgreSQL](https://www.postgresql.org/): Open-source relational database management system
- [PostgreSQL Docker Image](https://hub.docker.com/_/postgres): Official PostgreSQL image on Docker Hub
- [pgAdmin](https://www.pgadmin.org/): Web-based administration tool for PostgreSQL databases
- [pgAdmin Docker Image](https://hub.docker.com/r/dpage/pgadmin4): Official pgAdmin4 Docker image
- [Docker Networks](https://docs.docker.com/network/): How containers communicate with each other using Docker networks
- [Docker Volumes](https://docs.docker.com/storage/volumes/): Persistent storage mechanism for Docker containers
